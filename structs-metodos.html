Structs

Structs são tipos de dados compostos que agrupam campos relacionados. São a principal forma de criar tipos personalizados em Go.

Declaração Básica

structs_basico.go 
package main

import "fmt"

// Definindo uma struct
type Pessoa struct {
    Nome  string
    Idade int
    Email string
}

func main() {
    // Criando instância com valores
    p1 := Pessoa{
        Nome:  "João Silva",
        Idade: 25,
        Email: "joao@email.com",
    }
    
    // Criando com ordem dos campos
    p2 := Pessoa{"Maria Santos", 30, "maria@email.com"}
    
    // Struct vazia (valores zero)
    var p3 Pessoa
    
    // Acessando campos
    fmt.Println(p1.Nome)
    fmt.Println(p2.Idade)
    
    // Modificando campos
    p3.Nome = "Pedro Costa"
    p3.Idade = 28
    
    fmt.Printf("%+v\n", p1)  // Imprime com nomes dos campos
}

Structs Anônimas

structs_anonimas.go 
func main() {
    // Struct anônima
    pessoa := struct {
        Nome  string
        Idade int
    }{
        Nome:  "Ana",
        Idade: 22,
    }
    
    fmt.Println(pessoa.Nome)
    
    // Slice de structs anônimas
    usuarios := []struct {
        ID   int
        Nome string
    }{
        {1, "João"},
        {2, "Maria"},
        {3, "Pedro"},
    }
    
    for _, u := range usuarios {
        fmt.Printf("ID: %d, Nome: %s\n", u.ID, u.Nome)
    }
}

Ponteiros para Structs

structs_ponteiros.go 
type Conta struct {
    Numero int
    Saldo  float64
}

func main() {
    // Ponteiro para struct
    c1 := &Conta{
        Numero: 123,
        Saldo:  1000.00,
    }
    
    // Go permite acessar campos sem desreferenciar
    fmt.Println(c1.Saldo)  // ao invés de (*c1).Saldo
    
    // Modificando através do ponteiro
    c1.Saldo += 500
    fmt.Println(c1.Saldo)
    
    // Usando new
    c2 := new(Conta)
    c2.Numero = 456
    c2.Saldo = 2000.00
}
Dica: Go automaticamente desreferencia ponteiros ao acessar campos de struct, tornando o código mais limpo.

Métodos

Métodos são funções associadas a tipos específicos. Em Go, você pode definir métodos para qualquer tipo que você declare no mesmo pacote.

Métodos com Receiver

metodos_basico.go 
package main

import "fmt"

type Retangulo struct {
    Largura float64
    Altura  float64
}

// Método com value receiver
func (r Retangulo) Area() float64 {
    return r.Largura * r.Altura
}

// Método com value receiver
func (r Retangulo) Perimetro() float64 {
    return 2 * (r.Largura + r.Altura)
}

func main() {
    ret := Retangulo{
        Largura: 10,
        Altura:  5,
    }
    
    fmt.Printf("Área: %.2f\n", ret.Area())
    fmt.Printf("Perímetro: %.2f\n", ret.Perimetro())
}

Value Receiver vs Pointer Receiver

receivers.go 
type Contador struct {
    Valor int
}

// Value receiver - NÃO modifica o original
func (c Contador) IncrementarErrado() {
    c.Valor++  // Modifica apenas a cópia
}

// Pointer receiver - MODIFICA o original
func (c *Contador) Incrementar() {
    c.Valor++  // Modifica o original
}

// Pointer receiver
func (c *Contador) Decrementar() {
    c.Valor--
}

// Value receiver - apenas leitura
func (c Contador) ObterValor() int {
    return c.Valor
}

func main() {
    c := Contador{Valor: 0}
    
    c.IncrementarErrado()
    fmt.Println(c.Valor)  // 0 (não mudou)
    
    c.Incrementar()
    fmt.Println(c.Valor)  // 1
    
    c.Incrementar()
    c.Incrementar()
    fmt.Println(c.ObterValor())  // 3
}
Regra Importante: Use pointer receiver quando precisar modificar o receiver ou quando a struct for grande (para evitar cópias). Use value receiver para tipos pequenos e imutáveis.

Métodos em Tipos Customizados

metodos_tipos.go 
package main

import (
    "fmt"
    "strings"
)

// Tipo customizado baseado em string
type Texto string

// Métodos para o tipo Texto
func (t Texto) Maiuscula() string {
    return strings.ToUpper(string(t))
}

func (t Texto) Minuscula() string {
    return strings.ToLower(string(t))
}

func (t Texto) Tamanho() int {
    return len(t)
}

// Tipo customizado baseado em int
type Temperatura float64

func (t Temperatura) ParaFahrenheit() float64 {
    return (float64(t) * 9 / 5) + 32
}

func (t Temperatura) ParaKelvin() float64 {
    return float64(t) + 273.15
}

func main() {
    txt := Texto("Olá, Mundo!")
    fmt.Println(txt.Maiuscula())
    fmt.Println(txt.Minuscula())
    fmt.Println(txt.Tamanho())
    
    temp := Temperatura(25.0)
    fmt.Printf("%.2f°C = %.2f°F\n", temp, temp.ParaFahrenheit())
    fmt.Printf("%.2f°C = %.2fK\n", temp, temp.ParaKelvin())
}

Embedded Structs (Composição)

Go não tem herança, mas usa composição através de embedded structs para reutilizar código.

Embedding Básico

embedding.go 
package main

import "fmt"

type Endereco struct {
    Rua    string
    Numero int
    Cidade string
    Estado string
}

type Pessoa struct {
    Nome  string
    Idade int
    Endereco  // Embedded struct
}

func main() {
    p := Pessoa{
        Nome:  "João Silva",
        Idade: 30,
        Endereco: Endereco{
            Rua:    "Av. Principal",
            Numero: 123,
            Cidade: "São Paulo",
            Estado: "SP",
        },
    }
    
    // Acesso direto aos campos do embedded struct
    fmt.Println(p.Rua)     // Av. Principal
    fmt.Println(p.Cidade)  // São Paulo
    
    // Ou acesso explícito
    fmt.Println(p.Endereco.Estado)  // SP
}

Métodos em Embedded Structs

embedding_metodos.go 
type Animal struct {
    Nome string
}

func (a Animal) Respirar() {
    fmt.Printf("%s está respirando\n", a.Nome)
}

type Cachorro struct {
    Animal  // Embedding
    Raca string
}

func (c Cachorro) Latir() {
    fmt.Printf("%s está latindo: Au au!\n", c.Nome)
}

type Gato struct {
    Animal
    Cor string
}

func (g Gato) Miar() {
    fmt.Printf("%s está miando: Miau!\n", g.Nome)
}

func main() {
    dog := Cachorro{
        Animal: Animal{Nome: "Rex"},
        Raca:   "Labrador",
    }
    
    cat := Gato{
        Animal: Animal{Nome: "Mimi"},
        Cor:    "Branco",
    }
    
    // Métodos do embedded struct
    dog.Respirar()
    cat.Respirar()
    
    // Métodos próprios
    dog.Latir()
    cat.Miar()
}

Exemplos Práticos Completos

Sistema Bancário

banco.go 
package main

import (
    "fmt"
    "time"
)

type ContaBancaria struct {
    Numero      string
    Titular     string
    Saldo       float64
    DataCriacao time.Time
}

func NovaConta(numero, titular string, saldoInicial float64) *ContaBancaria {
    return &ContaBancaria{
        Numero:      numero,
        Titular:     titular,
        Saldo:       saldoInicial,
        DataCriacao: time.Now(),
    }
}

func (c *ContaBancaria) Depositar(valor float64) error {
    if valor <= 0 {
        return fmt.Errorf("valor deve ser positivo")
    }
    c.Saldo += valor
    fmt.Printf("Depósito de R$ %.2f realizado\n", valor)
    return nil
}

func (c *ContaBancaria) Sacar(valor float64) error {
    if valor <= 0 {
        return fmt.Errorf("valor deve ser positivo")
    }
    if valor > c.Saldo {
        return fmt.Errorf("saldo insuficiente")
    }
    c.Saldo -= valor
    fmt.Printf("Saque de R$ %.2f realizado\n", valor)
    return nil
}

func (c *ContaBancaria) Transferir(destino *ContaBancaria, valor float64) error {
    if err := c.Sacar(valor); err != nil {
        return err
    }
    return destino.Depositar(valor)
}

func (c ContaBancaria) ExibirSaldo() {
    fmt.Printf("Conta: %s | Titular: %s | Saldo: R$ %.2f\n", 
        c.Numero, c.Titular, c.Saldo)
}

func main() {
    conta1 := NovaConta("001", "João Silva", 1000.00)
    conta2 := NovaConta("002", "Maria Santos", 500.00)
    
    conta1.ExibirSaldo()
    conta2.ExibirSaldo()
    
    conta1.Depositar(500.00)
    conta1.Sacar(200.00)
    
    if err := conta1.Transferir(conta2, 300.00); err != nil {
        fmt.Println("Erro:", err)
    }
    
    fmt.Println("\nSaldos finais:")
    conta1.ExibirSaldo()
    conta2.ExibirSaldo()
}

Sistema de E-commerce

ecommerce.go 
package main

import "fmt"

type Produto struct {
    ID    int
    Nome  string
    Preco float64
}

type ItemCarrinho struct {
    Produto
    Quantidade int
}

func (i ItemCarrinho) Subtotal() float64 {
    return i.Preco * float64(i.Quantidade)
}

type Carrinho struct {
    Items []ItemCarrinho
}

func (c *Carrinho) Adicionar(produto Produto, quantidade int) {
    // Verifica se produto já existe
    for i, item := range c.Items {
        if item.ID == produto.ID {
            c.Items[i].Quantidade += quantidade
            fmt.Printf("%s: quantidade atualizada para %d\n", 
                produto.Nome, c.Items[i].Quantidade)
            return
        }
    }
    
    // Adiciona novo item
    c.Items = append(c.Items, ItemCarrinho{
        Produto:    produto,
        Quantidade: quantidade,
    })
    fmt.Printf("%s adicionado ao carrinho\n", produto.Nome)
}

func (c *Carrinho) Remover(produtoID int) {
    for i, item := range c.Items {
        if item.ID == produtoID {
            c.Items = append(c.Items[:i], c.Items[i+1:]...)
            fmt.Println("Produto removido")
            return
        }
    }
}

func (c Carrinho) Total() float64 {
    total := 0.0
    for _, item := range c.Items {
        total += item.Subtotal()
    }
    return total
}

func (c Carrinho) Exibir() {
    fmt.Println("\n=== CARRINHO DE COMPRAS ===")
    
    if len(c.Items) == 0 {
        fmt.Println("Carrinho vazio")
        return
    }
    
    for _, item := range c.Items {
        fmt.Printf("%s - R$ %.2f x %d = R$ %.2f\n",
            item.Nome, item.Preco, item.Quantidade, item.Subtotal())
    }
    
    fmt.Printf("\nTotal: R$ %.2f\n", c.Total())
}

func main() {
    carrinho := Carrinho{}
    
    p1 := Produto{1, "Mouse", 29.90}
    p2 := Produto{2, "Teclado", 89.90}
    p3 := Produto{3, "Monitor", 599.00}
    
    carrinho.Adicionar(p1, 2)
    carrinho.Adicionar(p2, 1)
    carrinho.Adicionar(p3, 1)
    carrinho.Adicionar(p1, 1)  // Adiciona mais um mouse
    
    carrinho.Exibir()
    
    carrinho.Remover(2)
    carrinho.Exibir()
}

Sistema de Gerenciamento de Funcionários

funcionarios.go 
package main

import "fmt"

type Funcionario struct {
    Nome      string
    Cargo     string
    Salario   float64
}

func (f Funcionario) ExibirInfo() {
    fmt.Printf("Nome: %s | Cargo: %s | Salário: R$ %.2f\n",
        f.Nome, f.Cargo, f.Salario)
}

type Gerente struct {
    Funcionario
    Departamento string
    Bonus        float64
}

func (g Gerente) SalarioTotal() float64 {
    return g.Salario + g.Bonus
}

func (g Gerente) ExibirInfo() {
    fmt.Printf("Nome: %s | Cargo: %s | Depto: %s | Salário Total: R$ %.2f\n",
        g.Nome, g.Cargo, g.Departamento, g.SalarioTotal())
}

type Desenvolvedor struct {
    Funcionario
    Linguagens []string
    Nivel      string
}

func (d Desenvolvedor) ExibirInfo() {
    fmt.Printf("Nome: %s | Cargo: %s | Nível: %s | Salário: R$ %.2f\n",
        d.Nome, d.Cargo, d.Nivel, d.Salario)
    fmt.Printf("Linguagens: %v\n", d.Linguagens)
}

func main() {
    gerente := Gerente{
        Funcionario: Funcionario{
            Nome:    "Ana Silva",
            Cargo:   "Gerente de TI",
            Salario: 8000.00,
        },
        Departamento: "Tecnologia",
        Bonus:        2000.00,
    }
    
    dev := Desenvolvedor{
        Funcionario: Funcionario{
            Nome:    "Carlos Santos",
            Cargo:   "Desenvolvedor",
            Salario: 6000.00,
        },
        Linguagens: []string{"Go", "Python", "JavaScript"},
        Nivel:      "Sênior",
    }
    
    fmt.Println("=== INFORMAÇÕES DOS FUNCIONÁRIOS ===")
    gerente.ExibirInfo()
    fmt.Println()
    dev.ExibirInfo()
}

Resumo e Boas Práticas

✅ Boas Práticas - Structs

  • Use PascalCase para nomes de structs exportadas
  • Use camelCase para structs privadas
  • Inicialize structs com valores nomeados para clareza
  • Agrupe campos relacionados em structs aninhadas
  • Use ponteiros para structs grandes para evitar cópias
  • Considere criar funções construtoras (New...)

✅ Boas Práticas - Métodos

  • Use pointer receiver quando precisar modificar o estado
  • Use pointer receiver para structs grandes
  • Seja consistente: se um método usa pointer, todos devem usar
  • Métodos de leitura podem usar value receiver
  • Nomeie métodos com verbos (Calcular, Processar, Obter)
  • Mantenha métodos pequenos e focados em uma responsabilidade

✅ Boas Práticas - Composição

  • Prefira composição sobre herança (Go não tem herança)
  • Use embedding para reutilizar comportamento
  • Seja explícito quando houver conflito de nomes
  • Documente relacionamentos entre tipos
  • Use interfaces para polimorfismo

⚠️ Erros Comuns

  • Esquecer de usar ponteiro quando precisa modificar
  • Misturar value e pointer receivers no mesmo tipo
  • Não inicializar campos de struct aninhada
  • Exportar campos que deveriam ser privados
  • Criar métodos muito grandes e complexos
  • Não validar dados no construtor
  • Copiar structs grandes por valor

💡 Quando Usar Cada Abordagem

  • Value Receiver: Tipos pequenos, operações de leitura, imutabilidade
  • Pointer Receiver: Modificação de estado, structs grandes, consistência
  • Structs: Agrupar dados relacionados, modelar entidades
  • Embedded Structs: Reutilizar campos e métodos, composição
  • Métodos: Comportamento associado a dados

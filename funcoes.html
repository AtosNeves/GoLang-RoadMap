Declaração de Funções

Funções são blocos de código reutilizáveis. Em Go, elas são declaradas com a palavra-chave func.

Sintaxe Básica

funcoes_basicas.go 
package main

import "fmt"

// Função sem parâmetros e sem retorno
func saudar() {
    fmt.Println("Olá, mundo!")
}

// Função com parâmetros
func saudarPessoa(nome string) {
    fmt.Println("Olá,", nome)
}

// Função com retorno
func somar(a int, b int) int {
    return a + b
}

// Parâmetros do mesmo tipo (forma curta)
func multiplicar(a, b int) int {
    return a * b
}

func main() {
    saudar()
    saudarPessoa("João")
    
    resultado := somar(5, 3)
    fmt.Println("Soma:", resultado)
    
    fmt.Println("Multiplicação:", multiplicar(4, 7))
}
💡 Dica: Quando múltiplos parâmetros consecutivos têm o mesmo tipo, você pode omitir o tipo até o último: func somar(a, b, c int)

Múltiplos Retornos

Uma das características mais poderosas do Go é a capacidade de retornar múltiplos valores de uma função.

Retornando Múltiplos Valores

multiplos_retornos.go 
package main

import "fmt"

// Retorna dois valores
func dividir(a, b float64) (float64, float64) {
    quociente := a / b
    resto := int(a) % int(b)
    return quociente, float64(resto)
}

// Retorna valor e erro (padrão Go)
func dividirSeguro(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("divisão por zero")
    }
    return a / b, nil
}

func main() {
    // Capturando múltiplos retornos
    quociente, resto := dividir(10, 3)
    fmt.Printf("Quociente: %.2f, Resto: %.0f\n", quociente, resto)
    
    // Ignorando um valor com _
    resultado, _ := dividir(20, 4)
    fmt.Println("Resultado:", resultado)
    
    // Tratando erro
    valor, err := dividirSeguro(10, 0)
    if err != nil {
        fmt.Println("Erro:", err)
    } else {
        fmt.Println("Valor:", valor)
    }
}

Retornos Nomeados

Go permite nomear os valores de retorno, o que cria variáveis automaticamente e permite retorno vazio.

retornos_nomeados.go 
package main

import "fmt"

// Retornos nomeados
func calculaRetangulo(largura, altura float64) (area, perimetro float64) {
    area = largura * altura
    perimetro = 2 * (largura + altura)
    return  // retorno vazio (naked return)
}

// Exemplo mais prático
func obterNomeCompleto(primeiro, ultimo string) (completo string, tamanho int) {
    completo = primeiro + " " + ultimo
    tamanho = len(completo)
    return
}

func main() {
    a, p := calculaRetangulo(5.0, 3.0)
    fmt.Printf("Área: %.2f, Perímetro: %.2f\n", a, p)
    
    nome, tam := obterNomeCompleto("João", "Silva")
    fmt.Printf("Nome: %s, Tamanho: %d\n", nome, tam)
}
⚠️ Atenção: Retornos vazios (naked returns) só devem ser usados em funções curtas. Em funções longas, eles tornam o código menos legível.

Funções Variádicas

Funções variádicas podem aceitar um número variável de argumentos. Use ... antes do tipo do último parâmetro.

Sintaxe Básica

variadicas.go 
package main

import "fmt"

// Função variádica simples
func somar(numeros ...int) int {
    total := 0
    for _, num := range numeros {
        total += num
    }
    return total
}

// Com parâmetros fixos e variádicos
func saudarVarios(saudacao string, nomes ...string) {
    for _, nome := range nomes {
        fmt.Printf("%s, %s!\n", saudacao, nome)
    }
}

func main() {
    // Chamando com quantidade variável de argumentos
    fmt.Println(somar(1, 2, 3))              // 6
    fmt.Println(somar(10, 20, 30, 40, 50))  // 150
    fmt.Println(somar())                    // 0
    
    // Expandindo slice com ...
    valores := []int{5, 10, 15}
    fmt.Println(somar(valores...))        // 30
    
    saudarVarios("Olá", "João", "Maria", "Pedro")
}

Exemplos Práticos

variadicas_praticas.go 
package main

import (
    "fmt"
    "strings"
)

// Calcula média de notas
func calcularMedia(notas ...float64) float64 {
    if len(notas) == 0 {
        return 0
    }
    
    soma := 0.0
    for _, nota := range notas {
        soma += nota
    }
    
    return soma / float64(len(notas))
}

// Encontra o maior valor
func maximo(nums ...int) int {
    if len(nums) == 0 {
        return 0
    }
    
    max := nums[0]
    for _, num := range nums {
        if num > max {
            max = num
        }
    }
    return max
}

// Concatena strings com separador
func concatenar(separador string, textos ...string) string {
    return strings.Join(textos, separador)
}

func main() {
    media := calcularMedia(7.5, 8.0, 9.5, 6.0)
    fmt.Printf("Média: %.2f\n", media)
    
    maior := maximo(45, 12, 89, 23, 67)
    fmt.Println("Maior valor:", maior)
    
    frase := concatenar(" - ", "Go", "Python", "JavaScript")
    fmt.Println(frase)
}

Tópicos Avançados

Funções como Valores

Em Go, funções são valores de primeira classe. Você pode atribuí-las a variáveis, passá-las como argumentos e retorná-las de outras funções.

funcoes_como_valores.go 
package main

import "fmt"

func main() {
    // Atribuindo função a uma variável
    somar := func(a, b int) int {
        return a + b
    }
    
    resultado := somar(5, 3)
    fmt.Println("Resultado:", resultado)
    
    // Função que recebe função como parâmetro
    executarOperacao(10, 5, somar)
    
    multiplicar := func(a, b int) int {
        return a * b
    }
    executarOperacao(10, 5, multiplicar)
}

func executarOperacao(a, b int, op func(int, int) int) {
    resultado := op(a, b)
    fmt.Println("Resultado da operação:", resultado)
}

Closures

Closures são funções anônimas que podem acessar variáveis do escopo externo.

closures.go 
package main

import "fmt"

// Função que retorna uma closure
func contador() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

// Gerador de multiplicadores
func multiplicador(fator int) func(int) int {
    return func(n int) int {
        return n * fator
    }
}

func main() {
    // Cada chamada mantém seu próprio estado
    incrementa1 := contador()
    incrementa2 := contador()
    
    fmt.Println(incrementa1())  // 1
    fmt.Println(incrementa1())  // 2
    fmt.Println(incrementa2())  // 1
    fmt.Println(incrementa1())  // 3
    
    // Multiplicadores diferentes
    dobro := multiplicador(2)
    triplo := multiplicador(3)
    
    fmt.Println(dobro(5))   // 10
    fmt.Println(triplo(5))  // 15
}

Defer

A palavra-chave defer adia a execução de uma função até que a função circundante retorne.

defer.go 
package main

import "fmt"

func exemplo() {
    defer fmt.Println("1 - Este será executado por último")
    defer fmt.Println("2 - Este será executado antes do 1")
    
    fmt.Println("Executando função normalmente")
}

// Uso prático: fechando arquivo
func lerArquivo(nome string) error {
    // Simulação
    fmt.Println("Abrindo arquivo", nome)
    defer fmt.Println("Fechando arquivo", nome)
    
    fmt.Println("Processando arquivo...")
    return nil
}

func main() {
    exemplo()
    fmt.Println()
    lerArquivo("dados.txt")
}
💡 Ordem de Execução: Múltiplos defers são executados em ordem LIFO (Last In, First Out) - último a entrar, primeiro a sair, como uma pilha.

Recursão

recursao.go 
package main

import "fmt"

// Fatorial usando recursão
func fatorial(n int) int {
    if n == 0 {
        return 1
    }
    return n * fatorial(n-1)
}

// Fibonacci usando recursão
func fibonacci(n int) int {
    if n <= 1 {
        return n
    }
    return fibonacci(n-1) + fibonacci(n-2)
}

// Soma de array com recursão
func somaArray(arr []int) int {
    if len(arr) == 0 {
        return 0
    }
    return arr[0] + somaArray(arr[1:])
}

func main() {
    fmt.Printf("Fatorial de 5: %d\n", fatorial(5))
    fmt.Printf("Fibonacci(10): %d\n", fibonacci(10))
    
    numeros := []int{1, 2, 3, 4, 5}
    fmt.Printf("Soma: %d\n", somaArray(numeros))
}

Sistema de Validação

validacao.go 
package main

import (
    "fmt"
    "strings"
)

func validarEmail(email string) (bool, string) {
    if email == "" {
        return false, "email não pode ser vazio"
    }
    
    if !strings.Contains(email, "@") {
        return false, "email deve conter @"
    }
    
    partes := strings.Split(email, "@")
    if len(partes) != 2 {
        return false, "formato de email inválido"
    }
    
    if partes[0] == "" || partes[1] == "" {
        return false, "email incompleto"
    }
    
    return true, "email válido"
}

func validarSenha(senha string) (valida bool, mensagens []string) {
    if len(senha) < 8 {
        mensagens = append(mensagens, "senha deve ter pelo menos 8 caracteres")
        return false, mensagens
    }
    
    temMaiuscula := false
    temMinuscula := false
    temNumero := false
    
    for _, char := range senha {
        switch {
        case char >= 'A' && char <= 'Z':
            temMaiuscula = true
        case char >= 'a' && char <= 'z':
            temMinuscula = true
        case char >= '0' && char <= '9':
            temNumero = true
        }
    }
    
    if !temMaiuscula {
        mensagens = append(mensagens, "senha deve conter letra maiúscula")
    }
    if !temMinuscula {
        mensagens = append(mensagens, "senha deve conter letra minúscula")
    }
    if !temNumero {
        mensagens = append(mensagens, "senha deve conter número")
    }
    
    valida = temMaiuscula && temMinuscula && temNumero
    if valida {
        mensagens = []string{"senha válida"}
    }
    
    return
}

func main() {
    // Testando validação de email
    emails := []string{"teste@gmail.com", "invalido", "@gmail.com"}
    
    for _, email := range emails {
        valido, msg := validarEmail(email)
        fmt.Printf("Email: %s - Válido: %t - %s\n", email, valido, msg)
    }
    
    fmt.Println()
    
    // Testando validação de senha
    senhas := []string{"Senha123", "fraca", "SemNumero"}
    
    for _, senha := range senhas {
        valida, msgs := validarSenha(senha)
        fmt.Printf("Senha: %s - Válida: %t\n", senha, valida)
        for _, msg := range msgs {
            fmt.Printf("  - %s\n", msg)
        }
    }
}

Calculadora Funcional

calculadora_funcional.go 
package main

import (
    "fmt"
    "math"
)

type Operacao func(float64, float64) float64

func calcular(a, b float64, op Operacao) float64 {
    return op(a, b)
}

func aplicarOperacoes(valores []float64, operacoes ...Operacao) []float64 {
    resultados := []float64{}
    
    for i := 0; i < len(valores)-1; i++ {
        for _, op := range operacoes {
            res := op(valores[i], valores[i+1])
            resultados = append(resultados, res)
        }
    }
    
    return resultados
}

func main() {
    // Definindo operações
    somar := func(a, b float64) float64 { return a + b }
    subtrair := func(a, b float64) float64 { return a - b }
    multiplicar := func(a, b float64) float64 { return a * b }
    dividir := func(a, b float64) float64 {
        if b == 0 {
            return 0
        }
        return a / b
    }
    potencia := func(a, b float64) float64 { return math.Pow(a, b) }
    
    // Usando as operações
    fmt.Println("10 + 5 =", calcular(10, 5, somar))
    fmt.Println("10 - 5 =", calcular(10, 5, subtrair))
    fmt.Println("10 * 5 =", calcular(10, 5, multiplicar))
    fmt.Println("10 / 5 =", calcular(10, 5, dividir))
    fmt.Println("2 ^ 3 =", calcular(2, 3, potencia))
    
    fmt.Println()
    
    // Aplicando múltiplas operações
    valores := []float64{10, 5, 2}
    resultados := aplicarOperacoes(valores, somar, multiplicar)
    
    fmt.Println("Resultados:", resultados)
}

Resumo e Boas Práticas

✅ Boas Práticas

  • Use nomes descritivos para funções (verbos ou ações)
  • Funções devem fazer uma coisa e fazê-la bem
  • Prefira múltiplos retornos ao invés de structs para valores simples
  • Sempre retorne erros como último valor de retorno
  • Use retornos nomeados apenas em funções curtas
  • Documente funções exportadas (públicas) com comentários
  • Use defer para garantir limpeza de recursos
  • Evite funções muito longas (máximo 50-100 linhas)
  • Limite o número de parâmetros (idealmente até 3-4)
  • Use funções variádicas quando fizer sentido aceitar múltiplos valores

⚠️ Erros Comuns

  • Ignorar valores de retorno (especialmente erros)
  • Usar naked returns em funções longas
  • Não documentar o que a função faz
  • Criar funções muito genéricas ou muito específicas
  • Esquecer que defer executa em ordem LIFO
  • Não validar entradas da função
  • Usar recursão sem caso base (causa stack overflow)
  • Modificar parâmetros slice/map sem documentar (side effects)

💡 Convenções de Nomenclatura

  • Funções públicas: Iniciam com letra maiúscula (PascalCase)
  • Funções privadas: Iniciam com letra minúscula (camelCase)
  • Getters: Não use "Get" no nome (ex: Name(), não GetName())
  • Setters: Use "Set" no nome (ex: SetName())
  • Conversores: Use "To" (ex: ToString())

Exercícios Práticos

🎯 Desafios

1. Calculadora Estatística

Crie funções variádicas para calcular média, mediana, moda e desvio padrão de uma série de números.

2. Sistema de Desconto

Crie uma função que retorna outra função (closure) para aplicar descontos. Ex: criarDesconto(10%) retorna função que aplica 10% de desconto.

3. Processador de Texto

Crie funções que recebam uma string e retornem: número de palavras, número de vogais, string invertida, e string com primeira letra de cada palavra maiúscula.

4. Sistema de Conversão

Crie um mapa de funções para converter unidades (temperatura, distância, peso). Use type para definir o tipo da função.

5. Gerador de Fibonacci

Crie uma função que retorna uma closure que gera números de Fibonacci sob demanda (cada chamada retorna o próximo número).

6. Pipeline de Funções

Crie uma função que aceita múltiplas funções e as executa em sequência, passando o resultado de uma para outra.

7. Cache com Closure

Implemente uma função que retorna uma closure que faz cache de resultados de cálculos caros (memoization).
